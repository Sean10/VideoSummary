会议纪要

会议时间： 2023年11月X日

会议地点： 远程会议

参会人员： Sage, Radek, Matt, Patrick, Kisi, Elaine, Mark 等

会议主题： Ceph 分布式存储项目进展及讨论

会议内容：

一、Ceph 存储性能优化

  1. 页面对齐问题： 针对ARM架构的页面对齐问题，讨论了64K分配和页面对齐的必要性，认为通常情况下对齐两页是为了直接IO，因此对齐操作是多余的。
  2. 日志优化： 讨论了Patrick提出的日志优化方案，该方案旨在避免过热，目前看起来已经准备就绪。
  3. 性能测试： 讨论了Mark提出的性能测试结果，认为需要进一步测试以验证其准确性。
  4. 加密功能： 讨论了Radek提出的加密功能，目前仅完成了单元测试，需要进一步测试集群运行结果。
  5. 跟踪器性能： 讨论了Tracker性能优化方案，认为这是一个短期性能提升方案，需要进一步讨论。

二、Ceph 数据库优化

  1. 缓冲区列表： 讨论了Sage提出的缓冲区列表优化方案，包括清理旧代码、优化编码方式等。
  2. 小向量实验： 讨论了Sage提出的小向量实验结果，认为该方案未能达到预期效果，需要进一步研究。
  3. 内存分配： 讨论了内存分配优化方案，包括使用SeaStar缓冲区原语、优化引用计数等。
  4. 键值存储： 讨论了Matt提出的键值存储优化方案，认为需要研究前缀树索引结构，并考虑在CStore中实现。

三、Ceph 代码维护

  1. 代码清理： 讨论了Sage提出的代码清理方案，包括清理旧代码、优化编码方式等。
  2. 代码合并： 讨论了Sage提出的代码合并方案，包括合并OP Tracker等。

四、后续行动计划

  1. 继续推进Ceph存储性能优化和数据库优化工作。
  2. 完成Ceph代码清理和合并工作。
  3. 研究前缀树索引结构，并在CStore中实现。
  4. 探索CStore在多种环境下的应用。

五、其他事项

  1. 讨论了Ceph项目在社区中的影响力。
  2. 讨论了Ceph项目的未来发展方向。

关键词： 页面对齐、日志优化、性能测试、加密功能、跟踪器、缓冲区列表、小向量实验、内存分配、键值存储、前缀树、CStore、代码清理、代码合并
