会议纪要

会议主题：Ceph Dashboard 团队 GSoC 实习生项目汇报

参会人员：Shreya Sharma（GSoC 实习生，Dashboard 团队）、Aryan（GSoC 实习生，Dashboard 团队）、其他团队成员

会议时间：[具体时间]

会议内容总结：

  1. Shreya Sharma 的项目汇报
    • 提出从 Ceph Dashboard 和 CLI 直接报告问题的功能，以提高用户报告问题的便利性和响应率。
    • 使用 Thor Requests 库，并通过 Ceph 问题跟踪器的 API 密钥进行身份验证。
    • 支持通过 CLI 和 Dashboard UI 报告问题，并计划支持附加图像、从文件中提供描述等功能。
  2. Aryan 的项目汇报
    • 开发 Ceph Dashboard 的视觉回归测试，以捕捉 CSS 错误和框架升级带来的视觉变化。
    • 使用 Amplitude 工具进行视觉回归测试。
    • 计划编写更多测试用例，完善 CI/CD 管道集成，并编写文档以帮助贡献者。
  3. Ceph Manager 性能优化
    • 针对 Manager 模块性能问题进行优化，包括 API 方法暴露、基准测试和多线程。
    • 使用 JSON 格式和缓存策略提高性能。
    • 计划进一步优化性能测试和数据格式。
  4. Jaeger 跟踪
    • 讨论使用 Jaeger 进行分布式跟踪的计划，以及如何通过跟踪来改进多站点和复杂操作的调试。
    • 计划实现可配置的跟踪选项，以适应不同场景。
  5. 崩溃报告
    • 介绍新的崩溃报告仪表板和崩溃报告机器人，用于跟踪和关联崩溃事件。
    • 计划进一步开发崩溃报告功能，以提高系统的可观察性和调试能力。

决定事项：

  • 继续推进视觉回归测试和问题报告功能的开发。
  • 探索更高效的性能测试和数据处理方法。
  • 开始实施 Jaeger 跟踪，并探索更高效的性能测试方法。
  • 开发新的崩溃报告功能。

后续行动计划：

  • Shreya Sharma 将继续完善问题报告功能，并探索附加图像和文件描述的支持。
  • Aryan 将继续编写更多视觉回归测试用例，并完善 CI/CD 集成。
  • 团队将开始实施 Jaeger 跟踪，并探索更高效的性能测试方法。
  • 开发新的崩溃报告功能。

其他备注：

  • 讨论了安全性和隐私问题，特别是如何在不牺牲安全性的前提下简化用户报告问题的流程。
  • 强调了自动化性能测试和数据处理方法的重要性，以及如何通过改进数据格式和缓存策略来提高性能。
