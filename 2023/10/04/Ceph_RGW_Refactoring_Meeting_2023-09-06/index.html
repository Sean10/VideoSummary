Ceph RGW 重构会议纪要 2023-09-06

2023-10-04
  • 视频总结

[改进后的中文总结内容]

本次会议主要讨论了Ceph RGW的重构议题，包括以下几个方面：

会议议题

  1. AWS SNS主题管理权限问题
    • 讨论了AWS SNS主题管理权限的分配问题，指出当前任何用户都可以创建、更新或删除主题，这与SNS文档中的描述不符。
    • 提出将主题与创建者用户关联，以确保只有创建者可以管理该主题。
  2. 多站点环境下的通知复制问题
    • 讨论了在多站点环境中，通知未被复制到次要站点的问题，这是一个已知的bug。
    • 提出将通知配置作为桶属性的可能性，以便通过现有的同步机制进行同步。
  3. AWS CLI工具的改进
    • 讨论了AWS CLI工具在获取和列出主题时应仅显示用户拥有的主题，而不是所有主题。
    • 提出了匿名主题的概念，即未与特定用户关联的主题。
  4. 主题与桶的关联及权限管理
    • 讨论了用户是否可以订阅其他用户创建的主题的问题。
    • 提出了使用策略来管理主题的访问权限。

决定事项

  1. 主题与用户关联
    • 决定将主题与创建者用户关联，并考虑现有部署的兼容性问题。
  2. 多站点环境下的通知复制
    • 确认了通知未被复制到次要站点的问题，并将继续跟踪和解决该问题。
  3. AWS CLI工具改进
    • 决定改进AWS CLI工具，使其仅显示用户拥有的主题。
  4. 主题与桶的关联及权限管理
    • 决定探索使用策略来管理主题的访问权限。

后续行动计划

  1. 主题与用户关联
    • 创建一个新的跟踪器，以继续讨论和实现主题与用户关联的功能。
  2. 多站点环境下的通知复制
    • 继续跟踪和解决通知未被复制到次要站点的问题。
  3. AWS CLI工具改进
    • 改进AWS CLI工具，使其仅显示用户拥有的主题。
  4. 主题与桶的关联及权限管理
    • 探索使用策略来管理主题的访问权限。

本次会议的讨论内容涵盖了Ceph RGW重构的多个关键方面，包括权限管理、多站点通知复制、CLI工具改进以及主题与桶的关联等问题，并制定了相应的后续行动计划。
