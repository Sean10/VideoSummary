Ceph 新读平衡器介绍会议由 IBM 的 RADOS 工程师 Laura Flores 主讲，主要讨论了 Ceph 分布式存储系统中新引入的读平衡器，旨在优化存储性能和读取效率。

会议要点：

  1. 读平衡的重要性：在分布式存储系统中，平衡读写请求对于优化性能至关重要。读平衡器专注于解决读取请求的不平衡问题，从而提高整体性能。

  2. 现有平衡器的局限：现有的容量平衡器主要处理写入请求，不处理读取请求，导致读取性能受限。

  3. 读平衡器的特点：适用于复制池，假设所有设备同质，操作成本低，仅涉及元数据操作，不涉及数据移动。

  4. 新命令和工具：引入了新的 ceph osd pool ls detail 命令，显示读取平衡分数。新增 ceph osd pg upmap primary 命令，用于重新分配 PG 的主 OSD。

  5. 未来计划和改进：计划将读平衡器集成到 Ceph 的测试套件中，作为默认平衡器模块的一部分。考虑将读平衡器扩展到异质系统。

  6. 问答环节：讨论了读平衡器在不同版本 Ceph 中的适用性、与自动缩放器的潜在冲突，以及网络延迟对读取平衡的影响。

决定事项：

  • 鼓励社区成员尝试新读平衡器，并提供反馈。
  • 计划将读平衡器集成到 Ceph 的默认平衡器模块中。

后续行动计划：

  • 完善文档，详细说明新工具的使用方法和效果。
  • 进行性能测试，验证读平衡器对集群性能的实际影响。
  • 持续改进读平衡器，考虑设备异质性的处理。
