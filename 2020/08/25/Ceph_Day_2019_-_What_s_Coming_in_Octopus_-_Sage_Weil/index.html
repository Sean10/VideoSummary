会议纪要

会议主题： Ceph 下一个版本 Octopus 的更新介绍 主讲人： Sage Weil 日期： 预计 2019 年 3 月发布

关键细节： 1. 发布周期调整： Ceph 项目调整为 12 个月的发布周期，每年 3 月发布一个主要版本。 2. Octopus 版本重点： - 可用性提升： 引入 Orchestrator API，支持 Rook 和 SSH Orchestrator，简化集群管理。 - 质量改进： 强化了 RADOS 的鲁棒性，改进对象恢复、快照和内存管理。 - 性能优化： 在 BlueStore 中进行性能改进，包括预取、压缩和内存管理。 - 多站点支持： 增强对多站点复制的支持，特别是在 RBD 和 CephFS 方面。 - 生态系统和社区集成： 加强与 Kubernetes 的集成，改进与 Samba 和 NFS 的集成。

讨论的主要议题： - 可用性和用户体验： 通过新的 API 和改进的仪表板提升用户体验。 - 质量和稳定性： 持续改进 Ceph 的质量和稳定性。 - 性能和扩展性： 通过优化存储后端和改进内存管理提高性能。 - 多站点和灾难恢复： 支持全球分布的数据管理和灾难恢复。

决定的事项： - 发布策略： 确定每年 3 月发布一个主要版本。 - Octopus 版本重点： 确认 Octopus 版本将重点关注可用性、质量、性能、多站点支持和生态系统集成。

后续行动计划： - 开发和测试： 继续开发和测试 Octopus 版本的新功能和改进。 - 文档更新： 更新文档以反映新的 Orchestrator API 和改进的安装步骤。 - 社区反馈： 鼓励社区成员提供反馈以帮助改进系统。

其他备注： - 社区和生态系统： 强调了与 Kubernetes 和其他社区项目的紧密合作。 - 未来展望： 讨论了 Ceph 的未来发展方向，包括对新硬件的支持和更高级的多站点功能。

会议结束： 会议在讨论了 Ceph 的未来发展方向和 Octopus 版本的重点后结束。
